# Feature Specification: サイドメニュー＋対戦モード選択

**Feature Branch**: `002-game-mode-select`  
**Created**: 2026-05-22  
**Status**: Draft  
**Input**: User description: "サイドメニュー作って、コンピュータ対戦と現在の人間同士を選択できるようにして"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 対戦モードを選んで対局を開始する (Priority: P1)

プレイヤーがサイドメニューを開き、「人間同士」または「コンピュータ対戦」を選択して対局を開始できる。

**Why this priority**: 本機能の核心。モード選択ができなければコンピュータ対戦も人間同士の継続も成立しない。

**Independent Test**: サイドメニューを開き、いずれかのモードを選択すると指定モードで新しい対局が始まることを確認できる。

**Acceptance Scenarios**:

1. **Given** アプリが表示されている状態で, **When** 画面左上のハンバーガーボタン（三本線アイコン）をクリックする, **Then** サイドメニューが開き「人間同士」と「コンピュータ対戦」の選択肢が表示される
2. **Given** サイドメニューで「人間同士」を選択し, **When** 対局開始を確定する, **Then** 現在と同様の2人対戦（ホットシート）モードで新しい対局が始まる
3. **Given** サイドメニューで「コンピュータ対戦」を選択し, **When** 対局開始を確定する, **Then** プレイヤーが先手・コンピュータが後手の対局が始まる
4. **Given** 対局中にモードを切り替えようとした場合, **When** モード変更を確定する, **Then** 確認の上で現在の対局を破棄し新しいモードで対局を再開する

---

### User Story 2 - コンピュータと対局する (Priority: P2)

コンピュータ対戦モードでは、プレイヤーが指し手を入力するとコンピュータが自動で応手する。

**Why this priority**: コンピュータ対戦モードを選んだ場合に実際にAIと対局できることがこの機能の主な価値。

**Independent Test**: コンピュータ対戦モードで対局を開始し、プレイヤーが指し手を入力するとコンピュータが応手することを確認できる。

**Acceptance Scenarios**:

1. **Given** コンピュータ対戦モードで対局中（後手番）, **When** プレイヤーが合法手を指す, **Then** コンピュータが自動で応手し、再びプレイヤーの番になる
2. **Given** コンピュータが応手を計算している間, **When** 計算が完了するまで, **Then** 盤面全体が半透明オーバーレイで覆われ中央に「思考中...」テキストが表示され、プレイヤーは操作できない状態（入力ロック）となる
3. **Given** コンピュータが詰みを受けた場合, **When** 詰みが検出される, **Then** 人間同士の対局と同様にプレイヤーの勝利が表示される

---

### User Story 3 - サイドメニューを開閉する (Priority: P3)

プレイヤーがいつでもサイドメニューを開閉でき、現在の対戦モードを確認できる。

**Why this priority**: 対局中にモードを確認・変更する手段を提供するUI要件。

**Independent Test**: 対局中にサイドメニューを開閉でき、現在のモードが表示されることを確認できる。

**Acceptance Scenarios**:

1. **Given** 対局中の状態で, **When** 左上のハンバーガーボタンをクリックする, **Then** サイドメニューが開閉し、対局画面は保持される
2. **Given** サイドメニューが開いている状態で, **When** メニュー外の領域をタップ/クリックする, **Then** サイドメニューが閉じる
3. **Given** サイドメニューが開いている状態で, **When** 現在のモードを確認する, **Then** 現在選択中のモードが強調表示で識別できる

---

### Edge Cases

- 対局中にモードを変更しようとした場合、進行中の対局が破棄される旨の確認を求める
- コンピュータの応手計算中に投了・リセット操作を行った場合、計算を中断して適切に処理する
- コンピュータが合法手を持たない（詰み）場合、自動的に投了として処理する

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは画面左上にハンバーガーボタン（三本線アイコン）を常時表示し、1回のクリックでサイドメニューをオーバーレイ表示/非表示できなければならない
- **FR-002**: サイドメニューは「人間同士（ホットシート）」と「コンピュータ対戦」の2つの対戦モード選択肢を表示しなければならない
- **FR-003**: プレイヤーが対戦モードを変更して確定すると、システムは新しいモードで対局を最初から開始しなければならない
- **FR-004**: 対局中にモードを変更しようとした場合、システムは確認ダイアログを表示し、了承されるまで対局を継続しなければならない
- **FR-005**: コンピュータ対戦モードでは、プレイヤーが指し手を入力した後にコンピュータが自動で応手しなければならない
- **FR-006**: コンピュータの応手計算中は、盤面全体を半透明オーバーレイで覆い中央に「思考中...」テキストを表示することでプレイヤーの入力をブロックしなければならない
- **FR-007**: コンピュータ対戦モードでは、プレイヤーは先手（下側）、コンピュータは後手（上側）として固定されなければならない
- **FR-008**: サイドメニューには現在選択中の対戦モードが視覚的に識別できる状態で表示されなければならない
- **FR-009**: 既存の「投了」「最初から」機能はコンピュータ対戦モードでも同様に動作し、「最初から」は現在の対戦モードを維持したまま盤面をリセットしなければならない

### Key Entities

- **対戦モード (GameMode)**: 人間同士 / コンピュータ対戦の区別
- **サイドメニュー (SideMenu)**: 開閉状態、現在選択中モード
- **コンピュータプレイヤー (ComputerPlayer)**: 応手ロジック、計算状態（待機中/計算中）

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: プレイヤーはサイドメニューからモードを選択して5秒以内に新しい対局を開始できる
- **SC-002**: コンピュータが応手するまでの待ち時間が5秒以内である（標準的な局面において）
- **SC-003**: コンピュータ対戦モードでプレイヤーが指し手を入力してから、コンピュータの応手が完了するまで誤操作（二重入力など）が発生しない
- **SC-004**: サイドメニューの開閉操作は1回のタップ/クリックで完結し、対局画面に影響を与えない
- **SC-005**: 将棋ルールを知っているユーザーが説明なしでコンピュータ対戦を開始・完了できる

## Clarifications

### Session 2026-05-22

- Q: サイドメニューの開閉UIはどのような形式か → A: 画面左上にハンバーガーボタン（三本線アイコン）を配置し、1クリックで開閉するオーバーレイ形式
- Q: コンピュータ思考中のUI表現はどうするか → A: 盤面全体を半透明オーバーレイで覆い、中央に「思考中...」テキストを表示する
- Q: 「最初から」ボタン押下時に対戦モードは保持されるか → A: 現在の対戦モードを維持したまま盤面をリセットする

## Assumptions

- コンピュータの強さは「合法手の中からランダムに選択する」最低限のレベルとする（難易度選択は対象外）
- プレイヤーは常に先手（下側）、コンピュータは常に後手（上側）に固定する（プレイヤーが後手を選ぶ機能は対象外）
- コンピュータの応手アルゴリズムは将棋の合法手を生成し、その中から選択する方式とする（評価関数・深い読みは対象外）
- サイドメニューはオーバーレイ形式（画面上に重なって表示）とし、対局画面の表示領域を変更しない
- 対局モード選択後は即座に対局が開始され、追加の設定画面は提供しない
- ネットワーク対戦・棋譜保存・対局時計は引き続き対象外とする
