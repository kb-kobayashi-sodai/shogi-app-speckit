## `/frontend-web-coding` スキル

### 呼び出しシグネチャ

```
/frontend-web-coding [実装依頼の説明]
```

**Input**: ユーザーからの自然言語による実装依頼（省略可）。省略した場合は会話の文脈から判断する。

### 事前ステップ（必須）

1. 以下の3ルールファイルを読み込む:
   - `@.claude/rules/frontend-web-directory.md`
   - `@.claude/rules/frontend-code-rule.md`
   - `@.claude/rules/frontend-web-react-rule.md`
2. 確認済みルール総数と主要項目（最大5点）を日本語で提示する

### Output（保証事項）

| 出力 | 形式 | 保証 |
|------|------|------|
| ルール確認サマリー | 「コーディングルール確認済み（計N項目）。重点：①...」 | 必ず先頭に表示 |
| 実装コード | TypeScript/TSX ファイル | ルール準拠、any型不使用、適切なディレクトリ配置 |
| チェックポイント確認 | 6項目のチェック結果（暗黙的） | SpecKit範囲・ディレクトリ・型安全・命名・React・マジックナンバー |

## `/frontend-web-review` スキル

### 呼び出しシグネチャ

```
/frontend-web-review [レビュー対象の指定]
```

**Input**: レビュー対象のファイルパス・コードスニペット・またはブランチ差分の説明（省略時は現在の変更全体を対象とする）。

### 事前ステップ（必須）

1. 以下の4ルールファイルを読み込む:
   - `@.claude/rules/frontend-web-directory.md`
   - `@.claude/rules/frontend-code-rule.md`
   - `@.claude/rules/frontend-web-react-rule.md`
   - `@.claude/rules/frontend-web-test-rule.md`
2. 確認済みルール総数と主要項目（最大5点）を日本語で提示する

### Output（保証事項）

| 出力 | 形式 | 保証 |
|------|------|------|
| ルール確認サマリー | 「レビュールール確認済み（計N項目）。重点：①...」 | 必ず先頭に表示 |
| レビュー結果セクション1 | ### 1. ディレクトリ構成 | 必ず存在 |
| レビュー結果セクション2 | ### 2. コーディングルール | 必ず存在 |
| レビュー結果セクション3 | ### 3. テストルール | 必ず存在 |
| レビュー結果セクション4 | ### 4. リファクタリング | 必ず存在 |
| リファクタリング実施 | コード変更 | 問題が見つかった場合は指摘のみでなく実際に修正する |

### リファクタリング判断基準

問題が見つかった場合は以下を実施する：
- ディレクトリ・ファイル命名の不備 → 修正提案（ファイル移動はユーザー確認後）
- `any` 型・型安全違反 → 即時修正
- React規約違反（クラスコンポーネント等）→ 即時修正
- テスト不足 → テストコードを追加
- リファクタリング候補（DRY・単一責務等）→ 即時実施
