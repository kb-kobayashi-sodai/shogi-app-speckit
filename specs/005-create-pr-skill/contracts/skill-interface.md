# Skill Interface Contract: speckit-git-pr

## コマンド

```
/speckit-git-pr [--draft] [--preview]
```

## 引数

| 引数 | 型 | 必須 | 説明 |
|------|----|------|------|
| --draft | flag | 任意 | ドラフト PR として作成する |
| --preview | flag | 任意 | PR 内容を表示するだけで実際には作成しない |

## 実行コンテキスト前提条件

- git コマンドが利用可能であること
- `gh` コマンド（GitHub CLI）がインストール済みであること
- カレントディレクトリが Git リポジトリ内であること
- フィーチャーブランチ（main 以外）にいること

## 出力フォーマット（成功時）

```
PR を作成しました: https://github.com/{owner}/{repo}/pull/{number}
```

## 出力フォーマット（プレビュー時）

```
## PR プレビュー

**タイトル**: {生成されたタイトル}

**本文**:
---
{生成された本文}
---

PR を作成するには --preview を外して再実行してください。
```

## PR 本文テンプレート

```markdown
## 概要
- {変更点の箇条書き（コミットメッセージから生成）}

## テスト計画
- [ ] {テスト項目（変更内容から推定）}

🤖 Generated with [Claude Code](https://claude.com/claude-code)
```

spec.md が存在する場合、`## 概要` にフィーチャー名と P1 ストーリー説明を追加する。

## エラー出力フォーマット

```
エラー: {エラーメッセージ}
```
