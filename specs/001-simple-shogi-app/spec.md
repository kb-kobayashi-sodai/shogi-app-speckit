# Feature Specification: シンプル将棋アプリ

**Feature Branch**: `001-simple-shogi-app`  
**Created**: 2026-05-22  
**Status**: Draft  
**Input**: User description: "シンプルな将棋アプリ作って"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 将棋の対局を開始・プレイする (Priority: P1)

プレイヤーが将棋盤を開き、2人でホットシート方式（同一デバイス）で将棋の対局を行う。駒を選択し合法手のみ指せる状態で交互に指し手を進め、詰みで勝敗が決まる。

**Why this priority**: 将棋アプリの核心機能。これなしには他の機能が意味を持たない。

**Independent Test**: アプリを開き、先手・後手で交互に指し手を進め、詰みまたは投了で対局終了を確認できれば単体でMVPとして成立する。

**Acceptance Scenarios**:

1. **Given** アプリを開いた状態で, **When** 対局開始ボタンを押す, **Then** 9×9の将棋盤と駒が初期配置された状態で先手番から対局が始まる
2. **Given** 先手番で, **When** 動かせる駒を1回クリック/タップする, **Then** その駒の合法手マスがハイライトされる
3. **Given** 合法手マスがハイライトされた状態で, **When** 移動先マスを1回クリック/タップする（2回目の操作）, **Then** 駒が移動し後手番に切り替わる
4. **Given** 駒が敵陣に入る/通過する移動をした場合, **When** 移動が確定する, **Then** 成り・不成りの選択ダイアログが表示される（強制的に成る場合を除く）
5. **Given** 王将が詰みになった, **When** 詰みが検出される, **Then** 対局終了と勝者が表示される

---

### User Story 2 - 取った駒を打つ (Priority: P2)

将棋の重要なルールである「持ち駒を盤上に打つ」操作を行う。取った駒を持ち駒エリアで選択し、合法な空きマスに打てる。

**Why this priority**: 将棋と他のチェス系ゲームを区別する根幹ルール。これがないと将棋として成立しない。

**Independent Test**: 対局中に駒を取り、持ち駒エリアから選択して合法マスに打てることを確認できる。

**Acceptance Scenarios**:

1. **Given** 駒を取った状態で, **When** 持ち駒エリアの駒をクリックする, **Then** 打てる合法マスがハイライトされる
2. **Given** 合法マスがハイライトされた状態で, **When** マスをクリックする, **Then** 駒が打たれ相手番に切り替わる
3. **Given** 二歩・打ち歩詰めなど非合法な場所, **When** 打とうとした場合, **Then** そのマスはハイライトされず打てない

---

### User Story 3 - 対局を最初からやり直す (Priority: P3)

対局途中または終了後に、新しいゲームを開始して盤面をリセットできる。

**Why this priority**: 誰でも気軽に遊べる体験に必要。プレイ継続率に直結する。

**Independent Test**: 対局中や終了後に「最初から」ボタンで初期配置に戻ることを確認できる。

**Acceptance Scenarios**:

1. **Given** 対局中または終了後, **When** 「最初から」ボタンを押す, **Then** 確認ダイアログが表示される
2. **Given** 確認ダイアログで確認した, **When** 確定操作を行う, **Then** 盤面が初期配置にリセットされ先手番から再開する

---

### User Story 4 - 投了する (Priority: P4)

現在の手番プレイヤーが不利を認め投了できる。

**Why this priority**: 詰みが来なくても対局を終了できる手段として必要。

**Independent Test**: 対局中に「投了」ボタンで勝敗表示が出ることを確認できる。

**Acceptance Scenarios**:

1. **Given** 対局中, **When** 「投了」ボタンを押す, **Then** 確認ダイアログが表示される
2. **Given** 確認後, **When** 投了が確定する, **Then** 相手プレイヤーの勝利が表示される

---

### Edge Cases

- 千日手（同一局面の繰り返し）: 同一局面が4回繰り返された時点でシステムが自動検出し、引き分けとして対局を自動終了する
- 行き所のない駒（例: 歩・香・桂を最奥段に打つ）の非合法手処理
- 王手放置の禁止（王手状態での移動で王が取られる手は非合法）
- 成駒を取った場合、持ち駒としては元の駒種に戻る

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは9×9の将棋盤を正しい駒の初期配置で表示しなければならない
- **FR-002**: システムは全ての将棋駒（王将/玉将・飛車・角行・金将・銀将・桂馬・香車・歩兵）とその成駒を正しく表現しなければならない
- **FR-003**: プレイヤーは駒を1回クリック/タップで選択し（1回目）、移動先を1回クリック/タップで確定する（2回目）クリック2回方式で操作できなければならない。ドラッグ&ドロップは不要。選択した駒の合法手のみハイライト表示される。
- **FR-004**: システムは将棋の全ルール（合法手判定・王手・詰み・打ち歩詰め・二歩・行き所のない駒など）を正しく適用しなければならない
- **FR-005**: プレイヤーは自分の手番でのみ操作できなければならない（相手の駒を動かせない）
- **FR-006**: 敵陣への移動/通過時に成れる駒は、成り・不成りの選択ができなければならない（強制的に成る場合は自動）
- **FR-007**: 取った駒は持ち駒として表示され、合法マスへ打てなければならない
- **FR-008**: 詰みが発生した時点でシステムが自動検出し、勝者を表示して対局を終了しなければならない
- **FR-011**: 同一局面が4回繰り返された時点でシステムが千日手を自動検出し、引き分けとして対局を自動終了しなければならない
- **FR-012**: 王手が発生した場合、王将のマスを赤くハイライトすると同時に「王手！」のテキスト通知を画面に表示しなければならない
- **FR-009**: プレイヤーはいつでも投了でき、相手の勝利として対局を終了できなければならない
- **FR-010**: プレイヤーは対局をいつでも最初からリセットできなければならない

### Key Entities

- **盤面 (Board)**: 9×9のマス目、各マスの状態（空き・駒の種類・所有者）
- **駒 (Piece)**: 駒の種類・所有者（先手/後手）・成り状態・各駒の移動ルール
- **持ち駒 (Captured Pieces)**: 各プレイヤーが取った駒の一覧（種類・枚数）
- **対局状態 (Game State)**: 現在の手番・王手状態・対局終了フラグ・勝者

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: プレイヤーはアプリを開いてから30秒以内に対局を開始できる
- **SC-002**: 駒を選択してから合法手のハイライトが即座（0.1秒以内）に表示される
- **SC-003**: 標準的な対局（約100手）を通じて、ルール違反の手が指せる状態にならない
- **SC-004**: 詰みが発生してから3秒以内に勝者表示がなされる
- **SC-005**: スマートフォン・タブレット・デスクトップブラウザのいずれでも操作可能である
- **SC-006**: 将棋のルールを知っているユーザーが、説明なしで対局を完了できる

## Clarifications

### Session 2026-05-22

- Q: 駒の操作方式（クリック2回 vs ドラッグ&ドロップ）はどちらを採用するか → A: クリック2回方式のみ（駒選択1回クリック → 移動先1回クリック）
- Q: 持将棋・入玉の扱い → A: スコープ外。入玉しても対局は続行し、詰みか投了まで続ける
- Q: 千日手の終局処理（自動 vs 手動宣言）→ A: 同一局面4回繰り返しで自動検出→自動引き分け終了
- Q: ホットシート時の盤面向き → A: 常に固定（先手が下・後手が上）、手番交代時に盤面は回転しない
- Q: 王手時の視覚フィードバック → A: 王将マスのハイライト（赤）＋「王手！」テキスト通知の両方

## Assumptions

- 対戦形式は同一デバイスでの2人対戦（ホットシート方式）とし、AIとの対戦は対象外とする
- ネットワーク対戦（オンライン対局）は対象外とする
- 棋譜の保存・読み込み機能は対象外とする（対局終了時に棋譜をリセット）
- 対局時計（持ち時間管理）は対象外とする
- 持将棋・入玉は対象外とする。入玉が発生しても対局は続行し、詰みまたは投了によってのみ終局する
- 初期配置は正式な将棋の配置（10枚の歩兵・2枚の香車・2枚の桂馬・2枚の銀将・2枚の金将・1枚の角行・1枚の飛車・1枚の王将）を使用する
- 成りについては、成れる場合は常に成り・不成りの選択肢を提示する（強制的に成らなければならない場合のみ自動）
- アプリはWebブラウザ上で動作し、インストール不要で利用できることを前提とする
- 盤面の向きは常に固定（先手が下・後手が上）とし、手番交代時に盤面は回転しない
